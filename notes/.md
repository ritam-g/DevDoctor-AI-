# 🩺 DevDoctor AI — Complete Project Summary

> An AI-powered code review platform that analyzes repositories, detects bugs, auto-fixes code, and lets developers collaborate — built with the full MERN stack in TypeScript.

---

## 🎯 What Problem Does It Solve?

Most developers push bad code to production because:
- No one reviews it properly
- Security vulnerabilities go unnoticed
- Performance issues are ignored
- Junior devs have no feedback loop

**DevDoctor AI** acts as an always-available senior engineer that reviews your entire codebase, explains what's wrong, and fixes it for you.

---

## 🧩 Core Modules & Features

### 📁 Module 1 — Repository Ingestion
- Upload a `.zip` file of your project
- Paste raw source code directly
- Connect a GitHub repository via OAuth
- Auto-detect project type: React / Node / MERN / Next.js / Python
- Generate a visual project structure map

### 🐛 Module 2 — AI Bug Detection
- 🔐 **Security issues** — `eval(userInput)`, plain-text passwords, weak JWT secrets
- ⚡ **Performance issues** — missing `useEffect` dependencies, N+1 queries, no pagination
- 📈 **Scalability issues** — 5000-line controllers, everything in `app.ts`, logic inside routes
- 🧹 **Best practices** — leftover `console.log`, missing `try/catch`, no input validation

### 🔧 Module 3 — Auto Fix (The Killer Feature)
- One-click **Fix** button per detected issue
- AI generates a before/after code diff
- User reviews the suggestion in a Monaco editor (VS Code in browser)
- One-click **Apply Fix** updates the file in the report
- Full fix history stored per repository

### 📖 Module 4 — Code Explainer
- Click **Explain this file** on any file in the project
- AI explains: purpose, flow, dependencies, known problems
- Great for onboarding or understanding legacy code

### 🏆 Module 5 — Architecture Score
- Generates a score card:
  ```
  Overall Score : 63 / 100
  Security      : 50
  Performance   : 70
  Scalability   : 60
  Code Quality  : 72
  ```
- Detailed explanation for each score
- Comparison across multiple submissions over time

### 💬 Module 6 — Community Discussions *(Planned Phase)*
- Make a project report **public**
- Other developers can view and comment on it
- Threaded discussions per file or per issue
- Great for open source projects and code reviews with teams

### 🎮 Module 7 — Practice Arena *(Planned Phase)*
- Paste badly written code
- AI generates a challenge: "Can you optimize this?"
- User rewrites the solution
- AI reviews and scores the improvement
- Like LeetCode but for real-world code quality

## 🔄 How It Works — Request Flow

```
User uploads repo
      ↓
API Gateway (auth check + rate limit)
      ↓
Project Service (reads + maps files)
      ↓
BullMQ Job pushed to Redis Queue
      ↓
Analysis Worker picks up job
      ↓
AI Service (LLM via LangChain)
      ↓
Report saved to MongoDB
      ↓
Socket.io emits progress to browser in real time
      ↓
User sees live: "Analyzing routes... Analyzing controllers... Done ✓"
```

## 🗃️ MongoDB Collections

- `users` — profile, auth, GitHub OAuth tokens
- `repositories` — uploaded projects, file maps, detected project type
- `reports` — full analysis results, scores, issue list
- `fixes` — before/after code, apply status, history per file
- `discussions` — threads, comments (public projects)
- `jobs` — BullMQ job tracking (status, progress, errors)
